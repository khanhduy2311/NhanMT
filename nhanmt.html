Matrix Chain Multiplication

Nhân ma trận dây chuyền tối ưu

Bài toán: Cho N ma trận A₁, A₂, ..., Aₙ. Nhập đầy đủ giá trị các phần tử trong mỗi ma trận. Tìm cách đặt dấu ngoặc tối ưu để số phép nhân vô hướng là ít nhất khi tính tích A₁ × A₂ × ... × Aₙ
(A ∗ B) ∗ C = A ∗ (B ∗ C) - Tính kết hợp
Lưu ý: Số cột của ma trận trước phải bằng số hàng của ma trận sau!

📖 Thuật toán — Matrix Chain Multiplication

Time: O(2ⁿ) Space: O(n) Đệ quy thuần

Phân rã bài toán thành các bài toán con nhỏ hơn bằng đệ quy. Với mỗi cặp (i,j), thử tất cả điểm cắt k và lấy min. Nhược điểm: tính lại nhiều lần nên chậm với n lớn.

MCM_DC(i, j): if i == j: return 0 // ma trận đơn, 0 phép nhân minCost = ∞ for k = i to j-1: cost = MCM_DC(i, k) + MCM_DC(k+1, j) + p[i-1] × p[k] × p[j] // chi phí nhân 2 cụm if cost < minCost: minCost = cost return minCost
Time: O(n³) Space: O(n²) DP Bottom-Up

Xây dựng bảng m[i][j] từ dưới lên — tính chuỗi độ dài 2 trước, rồi 3, ..., rồi n. Mỗi ô m[i][j] lưu số phép nhân tối thiểu để tính A[i..j].

MCM_BottomUp(p[0..n]): n = số ma trận m[i][i] = 0 ∀i // chuỗi độ dài 1 for len = 2 to n: // độ dài chuỗi for i = 1 to n-len+1: j = i + len - 1 m[i][j] = ∞ for k = i to j-1: // thử mọi điểm cắt cost = m[i][k] + m[k+1][j] + p[i-1]×p[k]×p[j] if cost < m[i][j]: m[i][j] = cost s[i][j] = k // lưu điểm cắt tối ưu return m[1][n]
Time: O(n³) Space: O(n²) Top-Down + Cache

Giống Chia để trị nhưng lưu kết quả vào bảng memo[i][j]. Trước khi tính, kiểm tra xem đã có kết quả chưa — nếu có thì dùng lại ngay (cache hit).

MCM_Memo(i, j): if i == j: return 0 if memo[i][j] ≠ -1: // cache hit! return memo[i][j] minCost = ∞ for k = i to j-1: cost = MCM_Memo(i, k) + MCM_Memo(k+1, j) + p[i-1]×p[k]×p[j] if cost < minCost: minCost = cost s[i][j] = k memo[i][j] = minCost // lưu vào cache return minCost
